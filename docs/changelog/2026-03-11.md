# Changelog — 11 марта 2026

## Paddle Billing Integration — полный цикл подписок

Реализована полная интеграция Paddle Checkout. Пользователи могут оформить подписку Pro/Premium прямо с лендинга. Webhook обновляет план в БД автоматически. Протестировано end-to-end: checkout открывается, оплата проходит, план меняется с Free → Pro.

**Файлы:**
- `components/billing/PaddleCheckout.tsx` — Paddle.js overlay с guard от stale событий (checkoutOpenedRef)
- `components/billing/AutoCheckout.tsx` — авто-открытие checkout после регистрации (?checkout=pro)
- `app/api/billing/create-checkout/route.ts` — серверный endpoint: auth + priceId validation + customData
- `app/api/billing/webhook/route.ts` — обработка subscription.activated/updated/canceled
- `lib/paddle/client.ts` — Paddle Node SDK client + getPlanFromPriceId helper
- `app/page.tsx` — кнопки Pro/Premium подключены к PaddleCheckout
- `app/register/page.tsx` — баннер выбранного плана + передача ?plan через emailRedirectTo
- `app/auth/callback/route.ts` — прокидывает ?plan в редирект на dashboard
- `app/dashboard/chat/page.tsx` — AutoCheckout монтируется для авто-открытия overlay

**Результат:**
- Checkout открывается без ошибок (successUrl устраняет transaction_default_checkout_url_not_set)
- 14 days free trial настроен в Paddle Sandbox
- Webhook верифицирует подпись (PADDLE_WEBHOOK_SECRET) и идемпотентно обновляет БД
- Незалогиненные → /register?plan=pro|premium → баннер плана → после верификации email авто-checkout
- Уже подписанные → редирект в /dashboard/settings

**Флоу незалогиненного пользователя:**
```
Лендинг "Get Premium" → /register?plan=premium → email verification
→ /auth/callback?plan=premium → /dashboard/chat?checkout=premium
→ AutoCheckout авто-открывает Paddle overlay
```
