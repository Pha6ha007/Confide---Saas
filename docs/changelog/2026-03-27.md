# 2026-03-27 — Security Hardening

## Security Headers

Добавлены все critical security headers в `next.config.js`:
- Content-Security-Policy (whitelist: Supabase, OpenAI, ElevenLabs, PostHog, Dodo)
- Strict-Transport-Security (HSTS, 2 года, preload)
- X-Frame-Options: DENY (anti-clickjacking)
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy (camera off, microphone self only)

**Файлы:** `next.config.js`

---

## Supabase Row Level Security (RLS)

Включён RLS на всех 16 таблицах. Неавторизованные запросы через Supabase REST API теперь возвращают 0 строк, вставка заблокирована.

- users, user_profiles, sessions, messages — только свои данные
- milestones — через проверку владельца goal (JOIN)
- knowledge_base — read-only для авторизованных
- safety_logs, rate_limits — полностью закрыты (только Prisma)

**Файлы:** `prisma/migrations/20260327060000_enable_rls_all_tables/migration.sql`

---

## IP Rate Limiting на публичных endpoints

Новый IP-based rate limiter для endpoints без авторизации:
- `/api/voice/preview` — 10 запросов/мин (защита ElevenLabs кредитов)
- `/api/contact` — 5 запросов/10 мин (защита от email спама)

**Файлы:** `lib/utils/ip-rate-limit.ts`, `app/api/voice/preview/route.ts`, `app/api/contact/route.ts`

**Файлы:** `lib/utils/safe-error.ts`, 11 route files

## GitHub Security CI

Настроен полный автоматический security pipeline:

- **Dependabot** — автоматические PR при уязвимостях (каждый понедельник)
- **CodeQL** — SAST сканирование кода на XSS, injection, path traversal (каждый push/PR)
- **Security CI** — npm audit + secret scan + security headers check + auth check + tsc (каждый push/PR)

**Файлы:** `.github/dependabot.yml`, `.github/workflows/codeql.yml`, `.github/workflows/security.yml`
